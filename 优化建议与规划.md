## 二、剩余问题（按严重程度排序）

### 2.1 🟢 全局函数污染（已解决）

**已完成**：
- 创建 `events.js` 统一事件绑定模块
- 移除所有 HTML `onclick` 属性，改为 `id` 属性
- 删除所有 `window.xxx` 全局函数挂载
- 使用 `addEventListener` 替代内联事件

---

### 2.2 � 硬编码常量分散（已解决）

**已完成**：
- 创建 `consts.js` 集中管理卡牌尺寸等常量
- CSS `:root` 中添加 `--card-width`、`--card-height` 等变量
- 所有 JS 文件使用 `CARD.WIDTH`、`CARD.HEIGHT` 常量

---

### 2.3 🟡 参数校验缺失（中优先级）

| 位置 | 问题 |
|------|------|
| logic.js checkReaction | `CARD_TEMPLATES[moved.templateId]` 未校验 undefined |
| engine.js renderAllCards | `CARD_TEMPLATES[card.templateId]` 未校验 undefined |
| engine.js recycleCard | `CARD_TEMPLATES[moved.templateId]` 未校验 undefined |

恶性 templateId 会直接导致 `Cannot read property 'name' of undefined` 白屏。

**建议**：在访问 `CARD_TEMPLATES[id]` 前添加防御检查。

---

### 2.4 🟡 拖拽事件作用域过大（中优先级）

`engine.js` 的 `processDrag` / `endDrag` 注册在 `document` 上，拖拽时整个页面都在监听。`endDrag` 已在 L288 先清理事件再判空，但 `processDrag` 仍然对全文档生效。

**建议**：将监听器绑定到 `boardCanvas`。


### 2.5 🟢 全量渲染仍需改进（低优先级）

拖拽时每次 `mousemove` 触发 `renderAllCards()`，遍历全部卡牌（当前 8 张影响小，但 20+ 张时会有明显卡顿）。之前增量渲染尝试因逻辑复杂回退。

**建议**：改为基于 `requestAnimationFrame` 的节流渲染。

---

### 2.6 🟢 components.css 过长（低优先级）

597 行未拆分，混合了卡牌、弹窗、进度条、对话等组件的样式。

**建议**：拆分为 `cards.css`、`modals.css`、`progress.css` 等。

---

### 2.7 🟢 探索并发竞态（低优先级）

`exploration.js` 的 `completeExploration` 在 1 秒后自动关闭，若用户同时手动关闭会导致不一致。

**建议**：添加关闭锁 `isClosing` 标志位。

## 四、建议执行顺序

```
立即: P1-4(全局函数消除) → P2-4(常量集中)
  ├── 这两个改动会触及 HTML 和多个 JS，改动范围最大，建议先做
  └── 完成后项目稳定性大幅提升
然后: P1-2(参数校验) → P1-3(事件作用域)
  ├── 防御性编程，无副作用
  └── 防止极少见的边界崩溃
最后: P2-2(节流渲染) → P4系列(体验增强)
  └── 锦上添花，按需选择
```
